👤 Persona Snapshot

📍 Who They Are

BA/PO team members (Cansel) designing the Sungur pricing module architecture. Focus on eliminating manual processes, unifying all fee requests through ACRM, and establishing service-based integrations.

🎯 Primary Goal

Design a unified pricing system where all fee requests flow through ACRM with direct service integrations to Sungur. Eliminate Subrates dependency and CSV-based manual uploads.

⚡ Key Challenges

Managing complex data structures (Ratecard → Rateline), ensuring currency consistency across systems, and designing the "most matching" parameter priority logic in the new platform.

📊 Success Metrics

100% ACRM request flow, zero CSV uploads, real-time ACRM-Sungur sync (vs. current 1-hour job), and elimination of duplicate fee line issues.

✅ Jobs to be Done

🔧 Subrates Elimination Architecture

Design direct ACRM → Sungur integration replacing the Excel → Subrates → CSV → COMIS flow. Build service-based APIs for fee uploads and validation.

📊 Fee Matching Logic Design

Implement "most matching" parameter priority for fee selection. Define hierarchy: Commodity → Handling Code → Handling Code Group. Manage Priority screen integration.

🔄 Real-time Sync Architecture

Replace hourly batch job with real-time GoldenGate → Kafka event streaming. Ensure ACRM and Sungur remain synchronized without accepted losses.

💱 Currency Management System

Design centralized currency mapping (USD→EUR) with station-based definitions. Ensure COMIS currency is master, with automated Subrates sync when updates occur.

📐 Ratecard/Rateline Data Model

Design Ratecard → Rateline hierarchical structure. Rateline ID as unique identifier. Replica tables for ACRM output without touching main COMIS tables.

🎯 Dynamic Pricing Integration

Design integration with RM teams for MCR/MMR/Bid Price calculations. Build coefficient-based pricing (Base Rate × Multiplier) for product variations.

🔧 Technical Challenges & Solutions

LAD Update Failures

Integration

If LAD cannot be updated during file upload, fee lines appear active and duplicate/clone in ACRM. Most issues stem from upload job failures.

Duplicate Line Prevention

Design

CSV-based flow can create duplicate fee lines in production. To-Be: Service integration with transactional guarantees.

ULD Type Pricing

Analysis Needed

Complex calculation: 1250kg × 2.7 + 250kg × 1.5 for AKE. Maindeck vs. lowerdeck variations require fresh analysis in Sungur design.

IATA Fare Upload

Integration

Current process involves manual DB operations + scripts. End date manipulation, new file date setting, then IATA file upload.

Currency Sync Priority

Design

COMIS must be up-to-date as master. If Subrates outdated, pricing team runs manual control process for currency alignment.

Flight Control for Future Bookings

Analysis Needed

1-month advance bookings may get old fees due to hourly sync. RM-side "flight control" logic needed in Sungur for accurate pricing.
