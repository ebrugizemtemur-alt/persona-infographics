🎯 OPERATIONS COORDINATOR (COCC) PERSONA

FLM - Flight Management Module | Cargo Flight Operations Coordination Key User Profile

"Son 3 gündeki operasyonel değişiklikleri yönetiyoruz - gecikme, iptal, diversiyon"
1
PERSONA SNAPSHOT
🎯
Operations Coordinator
Kargo Uçuş Operasyon Koordinasyon Şefliği (COCC)
PRIMARY GOAL
Manage real-time flight changes within the operational window (last 3 days): delays, cancellations, diversions, aircraft swaps, and adding new flights
KEY USERS
Sencer Tahmaz (COCC Operations)
DEPARTMENT
Kargo Pazarlama ve Satış Çözümleri Direktörlüğü - Uçuş Operasyon Koordinasyon Şefliği
OPERATIONAL ACTIONS
⏰ Delay ❌ Cancellation ↩️ Diversion ➕ Add Flight
2
JOBS TO BE DONE
⚠️ OPERATIONAL WINDOW: LAST 3 DAYS
COCC is authorized for flight changes within 3 days of departure. Beyond this, Cargo Schedule Directorate handles changes.
⏰ Flight Delay Management
Revise flight times via Atom. Enter details through Mona interface. Update flows to Netline and other systems.
❌ Flight Cancellation Decisions
Check load factors via CAP145/CAP147. Review cargo content including special cargo (live animals, perishables). Initiate cancellation for low-load flights.
✈️ Aircraft Type Changes
Handle aircraft swaps due to maintenance or operational needs. Coordinate with crew planning teams.
↩️ Diversions & Irregularities
Manage diversions due to weather (typhoons), operational issues, or pilot illness. Coordinate rapid response.
📋 Flight Change Portal Requests
Process requests from capacity units and stations via Individual Schedule Change Portal. Forward to authorized units based on date.

PAIN POINTS MATRIX
Pain Point Description Business Impact Severity
Multiple System Updates Required Changes in Atom must flow to Mona, then Netline, then other systems. Manual verification needed. Time-consuming; risk of inconsistent data across systems CRITICAL
Real-Time Pressure Operational changes require immediate action. No time for lengthy approval processes. Stress; potential for rushed decisions; need for rapid coordination CRITICAL
Special Cargo Considerations Must check for live animals, perishables before cancellation. Data scattered across screens. Risk of cancelling flights with critical cargo; customer impact HIGH
Authority Boundary Confusion 3-day operational window vs. Schedule Directorate responsibility not always clear Delayed decisions; miscommunication; duplicate efforts MEDIUM
Netline Plus Plus Transition Upcoming system change from current Netline to Netline Plus Plus next year Retraining needed; process changes expected MEDIUM
5
FLIGHT CHANGE WORKFLOW
1
Detect Issue / Receive Request
Identify operational issue (weather, mechanical, crew) or receive request via Portal
2
Check Flight Content
Open CAP145/CAP147 to view load factor, cargo content, special cargo types
3
Determine Action Type
Decide: Delay? Cancel? Add flight leg? Aircraft swap? Diversion?
4
Execute in Atom/Mona
Enter changes in Atom. Details flow through Mona interface to Netline.
5
Verify System Updates
Check that changes reflected correctly in Netline and downstream systems
6
Coordinate & Communicate
Notify affected stakeholders. Coordinate with crew planning if needed.
