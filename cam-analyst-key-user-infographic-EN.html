2
JOBS TO BE DONE
📊 Capacity Forecasting Analysis
Receive capacity forecasts from ACRM for passenger aircraft. Analyze no-show forecasts and overbooking capacity definitions. Evaluate forecast accuracy over time.
✍️ Manual Cargo Aircraft Capacity
For cargo (freighter) aircraft, enter capacity information manually in COMIS. Note: ACRM forecasting project for freighters is under development.
✅ Allotment Approval & Workflow
Review and validate allotment workflows. Evaluate practical scenarios for allotment approval and usage. Ensure edge cases are handled properly.
📋 Allotment Lifecycle Management
Manage allotment creation, updates, and adjustments during validity period. Coordinate between ACRM and COMIS for actions not supported in ACRM.
📈 Capacity vs Allotment Monitoring
Track allotment usage during booking and monitoring phases. Monitor partial utilization, remaining capacity visibility, and free-sale capacity interaction.

PAIN POINTS MATRIX
Pain Point Description Business Impact Severity
ACRM-COMIS Capability Gap Certain allotment configurations cannot be done in ACRM, requiring follow-up actions in COMIS Fragmented workflow between two systems; risk of missed actions CRITICAL
Freighter Manual Capacity Entry Cargo aircraft capacity must be entered manually in COMIS while ACRM only forecasts for pax aircraft Time-consuming; prone to errors; inconsistent data entry HIGH
Forecast Accuracy Tracking Difficulty in evaluating how accurate ACRM forecasts are over time Cannot improve forecasting models; capacity planning uncertainty MEDIUM
Allotment-Free Sale Interaction Complexity in understanding how allotment-controlled capacity interacts with free-sale capacity Suboptimal capacity utilization; customer allocation conflicts MEDIUM
ULD-Based Forecasting Gap No ULD-based forecasting approach available in current system Position-based capacity optimization limited; falls under ACRM team scope MEDIUM

ALLOTMENT LIFECYCLE WORKFLOW
1
Allotment Request (ACRM)

2
Approval Review
Evaluate request against available capacity, commercial rationale, and allotment type (Schedule vs Flight)
3
Allotment Creation
Create allotment in ACRM or COMIS depending on configuration requirements
4
COMIS Follow-up (If Needed)
Complete configurations not available in ACRM via COMIS
5
Monitor & Adjust
Track allotment usage, partial utilization, and adjust during validity period as needed
6
Expiry / Closure
Allotment expires or is released back to free-sale capacity

7
ALLOTMENT KEY CONCEPTS
SCHEDULE ALLOTMENT
Long-term capacity allocation based on flight schedules; applies across multiple flights on a route
FLIGHT ALLOTMENT
Specific capacity reserved for a single flight instance
HARD ALLOTMENT
Guaranteed capacity; must be used or explicitly released
SOFT ALLOTMENT
Priority capacity; can be reclaimed if not used by cutoff
ALLOTMENT TARGETS
Customer Groups Station Groups Specific Routes
